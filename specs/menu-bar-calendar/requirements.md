# 요구사항 문서

## 소개

MiniCalendar는 macOS 메뉴 막대에서 실행되는 간단한 캘린더 애플리케이션입니다. 메뉴 막대에 현재 날짜와 시간을 표시하고, 클릭 시 월간 달력을 팝업으로 보여줍니다. 이벤트 관리 기능 없이 순수하게 날짜/시간 확인과 달력 보기 기능만 제공하는 경량 앱입니다.

## 요구사항

### 요구사항 1: 메뉴 막대 표시

**User Story:** 사용자로서, 메뉴 막대에서 현재 날짜와 시간을 항상 확인할 수 있어야 합니다.

#### Acceptance Criteria

1. WHEN 앱이 실행되면 THEN 시스템은 macOS 메뉴 막대에 날짜와 시간을 표시해야 합니다.
2. WHEN 시간이 변경되면 THEN 시스템은 메뉴 막대의 시간 표시를 실시간으로 업데이트해야 합니다.
3. WHEN 날짜가 변경되면 THEN 시스템은 메뉴 막대의 날짜 표시를 자동으로 업데이트해야 합니다.
4. IF 사용자가 "시간 표시" 옵션을 비활성화하면 THEN 시스템은 메뉴 막대에서 시간을 숨겨야 합니다.
5. IF 사용자가 "날짜 표시" 옵션을 비활성화하면 THEN 시스템은 메뉴 막대에서 날짜를 숨겨야 합니다.
6. IF 사용자가 "아이콘만 표시" 옵션을 활성화하면 THEN 시스템은 캘린더 아이콘만 표시해야 합니다.

---

### 요구사항 2: 달력 팝업

**User Story:** 사용자로서, 메뉴 막대를 클릭하면 현재 월의 달력을 볼 수 있어야 합니다.

#### Acceptance Criteria

1. WHEN 사용자가 메뉴 막대 아이템을 클릭하면 THEN 시스템은 달력 팝업을 표시해야 합니다.
2. WHEN 달력 팝업이 표시되면 THEN 시스템은 현재 월의 전체 달력을 보여줘야 합니다.
3. WHEN 달력 팝업이 표시되면 THEN 시스템은 오늘 날짜를 시각적으로 강조 표시해야 합니다.
4. WHEN 사용자가 이전/다음 월 버튼을 클릭하면 THEN 시스템은 해당 월의 달력을 표시해야 합니다.
5. WHEN 사용자가 달력 외부 영역을 클릭하면 THEN 시스템은 달력 팝업을 닫아야 합니다.
6. WHEN 사용자가 달력의 특정 날짜를 클릭하면 THEN 시스템은 아무 동작도 수행하지 않아야 합니다 (보기 전용).

---

### 요구사항 3: 설정 화면

**User Story:** 사용자로서, 날짜/시간 표시 형식과 달력 옵션을 설정할 수 있어야 합니다.

#### Acceptance Criteria

1. WHEN 사용자가 설정 메뉴를 열면 THEN 시스템은 설정 창을 표시해야 합니다.

**일반 설정:**
2. WHEN 사용자가 "로그인 시 실행" 옵션을 활성화하면 THEN 시스템은 macOS 로그인 시 자동으로 앱을 시작해야 합니다.
3. WHEN 사용자가 "클래식 달력 모양" 옵션을 변경하면 THEN 시스템은 달력 UI 스타일을 변경해야 합니다.
4. WHEN 사용자가 "아이콘만 표시" 옵션을 활성화하면 THEN 시스템은 메뉴 막대에 캘린더 아이콘만 표시해야 합니다.

**시간 옵션:**
5. WHEN 사용자가 "시간 표시" 옵션을 변경하면 THEN 시스템은 메뉴 막대의 시간 표시 여부를 변경해야 합니다.
6. WHEN 사용자가 "초 단위 표시" 옵션을 활성화하면 THEN 시스템은 시간에 초를 포함하여 표시해야 합니다.
7. WHEN 사용자가 "24시간제 사용" 옵션을 활성화하면 THEN 시스템은 24시간 형식으로 시간을 표시해야 합니다.
8. IF "24시간제 사용" 옵션이 비활성화되어 있고 사용자가 "AM/PM 표시" 옵션을 활성화하면 THEN 시스템은 시간과 함께 AM/PM을 표시해야 합니다.

**날짜 옵션:**
9. WHEN 사용자가 "요일 표시" 옵션을 활성화하면 THEN 시스템은 메뉴 막대에 요일을 포함하여 표시해야 합니다.
10. WHEN 사용자가 "날짜 표시" 옵션을 활성화하면 THEN 시스템은 메뉴 막대에 날짜를 표시해야 합니다.
11. WHEN 사용자가 날짜 형식을 변경하면 THEN 시스템은 지정된 형식으로 날짜를 표시해야 합니다.

**달력 옵션:**
12. WHEN 사용자가 "주 시작 요일" 설정을 변경하면 THEN 시스템은 달력의 첫 번째 열을 해당 요일로 시작해야 합니다.
13. IF 사용자가 "일요일 시작"을 선택하면 THEN 달력은 일요일부터 토요일 순서로 표시되어야 합니다.
14. IF 사용자가 "월요일 시작"을 선택하면 THEN 달력은 월요일부터 일요일 순서로 표시되어야 합니다.

---

### 요구사항 4: 앱 생명주기

**User Story:** 사용자로서, 앱이 백그라운드에서 안정적으로 실행되어야 합니다.

#### Acceptance Criteria

1. WHEN 앱이 시작되면 THEN 시스템은 Dock에 아이콘을 표시하지 않아야 합니다 (메뉴 막대 전용 앱).
2. WHEN 사용자가 "종료" 버튼을 클릭하면 THEN 시스템은 앱을 완전히 종료해야 합니다.
3. WHEN 앱이 종료되면 THEN 시스템은 현재 설정을 저장해야 합니다.
4. WHEN 앱이 다시 시작되면 THEN 시스템은 이전에 저장된 설정을 복원해야 합니다.
5. WHEN macOS 시스템이 다크 모드로 변경되면 THEN 시스템은 앱 UI를 다크 모드에 맞게 조정해야 합니다.

---

### 요구사항 5: 기술 요구사항

**User Story:** 개발자로서, 앱이 macOS 네이티브 환경에서 최적화되어 실행되어야 합니다.

#### Acceptance Criteria

1. 앱은 Swift 언어로 개발되어야 합니다.
2. 앱은 macOS 13.0 (Ventura) 이상을 지원해야 합니다.
3. 앱은 SwiftUI 또는 AppKit을 사용하여 네이티브 UI를 구현해야 합니다.
4. 앱은 시스템 리소스를 최소한으로 사용해야 합니다 (백그라운드 실행 최적화).
5. 설정 데이터는 UserDefaults를 사용하여 로컬에 저장되어야 합니다.
